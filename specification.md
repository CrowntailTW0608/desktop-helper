## 1. 概述

一個 Python 桌面小幫手，概念類似過去 Microsoft Office 的迴紋針小幫手（Clippy）。
主體是一個懸浮於桌面的圓形視窗，內容顯示使用者自訂的 GIF 動畫；點擊後在周圍展開數個衛星圓圈，作為應用程式的快速啟動器。
主圓圈外圍以三層圓環常駐顯示 Claude 用量（Session／Weekly／特定模型），一瞥即知用量狀態。

### 1.1 目標

- 提供一個可愛、可自訂外觀的桌面常駐小工具。
- 作為應用程式快速啟動器：點一下主圓圈、再點衛星圓圈即可開啟常用程式。
- 常駐顯示 Claude 用量三指標（整合自 claude-usage-widget 專案），不用開瀏覽器就能掌握用量。
- 所有自訂項目（GIF、應用程式連結、Claude 憑證）皆可透過內建設定視窗完成，不需手動改設定檔。

### 1.2 非目標（Non-Goals）

- 不做 AI 對話、語音、提醒等助理功能。
- 不做跨平台支援；第一版僅支援 Windows 10/11。
- 不做多組設定檔（profile）切換。
- 不做衛星圓圈巢狀分層（衛星底下再開子選單）。
- Claude 用量僅顯示三個百分比與 reset 倒數；不納入原 widget 的事故通報（incidents）、透明度調整、完整/精簡模式切換。

## 2. 使用者情境（User Stories)

| 編號 | 情境 |
|------|------|
| US-1 | 使用者啟動程式後，桌面出現一個播放 GIF 的圓形小幫手，永遠置頂。 |
| US-2 | 使用者點擊主圓圈，周圍展開衛星圓圈；點擊某個衛星即開啟對應的應用程式。 |

## 3. 功能需求（Functional Requirements）

### 3.1 主圓圈（Main Bubble）

- **FR-1** 主視窗為無邊框、背景透明的圓形視窗，僅顯示圓形區域內容。
- **FR-2** 圓圈內顯示使用者自訂的 GIF 動畫（循環播放）；GIF 以圓形遮罩裁切並縮放至圓圈大小。
- **FR-2a** GIF 來源（`gif_path`）可為單一 `.gif` 檔或資料夾，依內容自動判別：
  - 檔案：固定顯示該 GIF。
  - 資料夾：輪播其中所有 `.gif`，每次切換隨機挑選（不連續重複同一張）；間隔可設定（`gif_interval`，秒，預設 60）。每次切換時重新掃描資料夾，新增檔案即自動加入輪播。
- **FR-3** 未設定 GIF、路徑失效或資料夾內無 `.gif` 時，顯示內建預設圖案（純色圓 + 簡單圖示）。
- **FR-4** 主視窗永遠置頂（always on top）。
- **FR-5** 主視窗不出現在工作列（taskbar）。
- **FR-6** 按住主圓圈可拖曳移動；放開後位置寫入設定檔，下次啟動還原。
- **FR-7** 單擊主圓圈（非拖曳）切換衛星圓圈的展開/收合。

### 3.2 衛星圓圈（Satellites）

- **FR-8** 展開時，衛星圓圈以主圓圈為中心、等角度環形排列；展開狀態下拖曳主圓圈，衛星即時跟隨移動。
- **FR-9** 衛星內容 = 使用者設定的應用程式連結（每個顯示該程式的圖示或名稱縮寫）+ 固定一個「設定」齒輪衛星。
- **FR-10** 應用程式連結數量：0～8 個，由使用者在設定中增減（含齒輪最多 9 個衛星）。
- **FR-11** 點擊應用程式衛星：開啟該連結目標，啟動後自動收合衛星。支援四種目標：
  - 執行檔／捷徑（`.exe`、`.lnk` 等）：等同於在檔案總管雙擊。
  - 資料夾：以檔案總管開啟。
  - 網址（`http://`、`https://`）：以預設瀏覽器開啟。
  - 帶參數的指令列（例：`"C:\...\chrome.exe" --profile-directory="Profile 7"`）：執行檔路徑需以雙引號括住，後接參數；以子行程啟動，不經過 shell。判別順序：先比對 URL 前綴、再檢查是否為現有資料夾／檔案，都不符合才嘗試解析成「引號執行檔＋參數」；若首個引號內路徑不是有效檔案，視為連結失效（FR-14）。
- **FR-12** 點擊齒輪衛星：開啟設定視窗，並收合衛星。
- **FR-13** 衛星展開狀態下，點擊主圓圈或衛星以外的區域（或再點一次主圓圈）即收合。
- **FR-14** 連結目標失效（檔案或資料夾不存在）時，顯示錯誤提示，不得閃退。

### 3.3 設定視窗（Settings）

- **FR-15** 設定視窗為一般標準視窗（有標題列、可關閉），非圓形。
- **FR-16** GIF 設定：可以檔案對話框挑選 `.gif` 檔，或以資料夾對話框挑選輪播資料夾（FR-2a），並可設定輪播間隔秒數；選定後主圓圈立即套用；提供「還原預設」。
- **FR-17** 應用程式連結清單：列表顯示現有連結（名稱 + 目標），提供新增、編輯、刪除，並可拖曳調整順序（順序即衛星排列順序，見 5.1 節）。
  - 新增/編輯欄位：顯示名稱、目標（執行檔／捷徑／資料夾以選擇對話框挑選，網址與任意路徑允許手動輸入）。
- **FR-18** 「開機時自動啟動」開關（透過 Windows 使用者登錄檔 `HKCU\...\Run` 實作）。
- **FR-19** 設定變更即時生效並寫入設定檔，無需重啟程式。
- **FR-20** Claude 用量設定區：「啟用用量顯示」開關、Session Key 與 Org ID 輸入欄位。未啟用或未填憑證時不顯示圓環、不發出任何網路請求。

### 3.4 系統匣（System Tray）

- **FR-21** 程式執行期間，通知區顯示系統匣圖示。
- **FR-22** 右鍵選單至少包含：顯示/隱藏小幫手、開啟設定、結束程式。
- **FR-23** 「結束程式」為唯一正常關閉途徑（主圓圈本身無關閉鈕）。

### 3.5 Claude 用量顯示（Usage Rings）

資料來源與取得邏輯沿用 `D:\2. pyData\1.Project\claude-usage-widget` 專案（`claude_usage_utils.py` 的 `fetch_usage`），呼叫 `https://claude.ai/api/organizations/{org_id}/usage`，以 Session Key cookie 驗證。

- **FR-24** 啟用時，主圓圈外圍顯示三層同心圓環，由內而外依序為：
  1. Current Session（5hr）
  2. 特定模型 Weekly（7d，名稱取自 API `scope.model.display_name`，例如 Opus）
  3. Weekly Limit（7d）
- **FR-25** 每層圓環以弧長表示用量百分比（12 點鐘方向起、順時針），顏色依 API `severity`：normal 綠、warning 橘、critical 紅；未使用部分以深灰色底環顯示。
- **FR-25a** 每層圓環在「時間經過百分比」位置畫一條白色基準線，代表若均勻消耗用量應到達的位置，供與實際用量弧比對（例：5hr 視窗已經過 1hr，白線在 20% 處）。時間視窗：session 5hr、weekly 皆 7d，經過比例 =（現在 − 視窗起點）/ 視窗長度，視窗起點 = reset 時間往前推一個視窗。
- **FR-26** 每 60 秒於背景執行緒自動更新一次，不得阻塞 UI（GIF 動畫、點擊、拖曳不受影響）。
- **FR-27** 滑鼠懸停主圓圈時，顯示浮動資訊卡：三項名稱、精確百分比、各自的 reset 倒數（沿用原專案 `format_reset_time` 格式）；在主圓圈按右鍵立即顯示同一資訊卡，不需等待懸停延遲。
- **FR-28** 取得失敗（401/403、網路錯誤等）時：先容忍暫時性錯誤——保留並繼續顯示上次成功的資料，連續失敗達 3 次（約 3 分鐘）才將三環轉灰、懸停資訊卡顯示錯誤原因；401/403 屬暫時性或金鑰失效皆有可能，訊息不斷言「已過期」。不得閃退，每 60 秒持續自動重試，成功即復原。

## 4. 非功能需求（Non-Functional Requirements）

- **NFR-1** 平台：Windows 10/11；Python 3.14（開發環境為 3.14.6，PySide6 6.11 支援範圍 >=3.10, <3.15）。
- **NFR-2** GUI 框架：PySide6（LGPL）。
- **NFR-3** 常駐記憶體佔用以輕量為原則；GIF 動畫使用 QMovie 原生播放，不自行逐幀處理。
- **NFR-5** 相依套件僅 PySide6 與 requests（用量 API 呼叫）。
- **NFR-6** Session Key 為敏感資訊：僅存於本機設定檔，不寫入任何 log；設定視窗的輸入欄位以密碼模式（遮罩）顯示。
- **NFR-4** 設定檔毀損或缺失時，以預設值啟動並重建設定檔，不得閃退。

## 5. 資料與設定

### 5.1 設定檔

- 位置：`%APPDATA%\desktop-helper\config.json`
- 格式（示意）：

```json
{
  "gif_path": "C:/Users/xxx/Pictures/helper.gif",
  "gif_interval": 60,
  "position": { "x": 1200, "y": 640 },
  "auto_start": false,
  "links": [
    { "name": "VS Code", "path": "C:/Users/xxx/AppData/Local/Programs/Microsoft VS Code/Code.exe" },
    { "name": "專案資料夾", "path": "D:/2. pyData/1.Project" },
    { "name": "GitHub", "path": "https://github.com" }
  ],
  "claude_usage": {
    "enabled": true,
    "session_key": "sk-ant-sid01-...",
    "org_id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
  }
}
```

- `links` 順序即衛星排列順序（齒輪固定為最後一個衛星）。
- `path` 依內容自動判別類型：`http://`／`https://` 開頭為網址、指向資料夾為資料夾、其餘為執行檔／捷徑，不另設 type 欄位。
- `gif_path` 同樣自動判別：檔案＝單張、資料夾＝輪播（FR-2a）。

## 6. UI 規格

### 6.1 主圓圈

- 預設直徑 96 px（第一版固定，不開放調整）。
- GIF 置中、cover 縮放、圓形遮罩。

### 6.2 用量圓環

- 三層同心圓環緊貼主圓圈外圍：每環寬約 3 px、環距 2 px，總外徑約 120 px（主視窗需預留此空間）。
- 由內而外：Session（5hr）→ 模型 Weekly（7d）→ Weekly（7d）。
- 弧線圓端（round cap），12 點鐘起順時針；底環深灰、進度弧依 severity 上色（綠 `#44cc66`、橘 `#f0a030`、紅 `#ff4444`，沿用原專案配色）。
- 均勻消耗基準線：白色、寬 1.5 px 的徑向短線，跨越該環環寬（FR-25a）。
- 懸停資訊卡：深色小卡，每行「名稱、百分比、reset 倒數」，錯誤時顯示錯誤訊息；跟隨主圓圈位置顯示於旁側。
- 用量顯示停用時，三環完全不繪製，主圓圈外觀與原設計相同。

### 6.3 衛星圓圈

- 直徑 48 px，距主圓圈中心約 90 px，等角度環形分佈。
- 展開/收合必須有動畫（必要功能）：衛星自主圓圈中心向外飛出並淡入，收合時反向；時長約 150～250 ms。
- 應用程式衛星顯示目標的系統圖示（執行檔圖示、資料夾圖示）；網址顯示內建地球圖示；取不到圖示時顯示名稱前 1～2 個字。
- 齒輪衛星使用內建齒輪圖示。
- 滑鼠懸停衛星時顯示 tooltip（連結名稱）。

### 6.4 設定視窗

- 單頁式，由上而下四區塊：
  1. **外觀**：GIF 路徑顯示 + 「選擇 GIF」「選擇資料夾」「還原預設」按鈕 + 輪播間隔（秒）欄位。
  2. **應用程式連結**：清單 + 「新增」「編輯」「刪除」按鈕。
  3. **Claude 用量**：「啟用用量顯示」核取方塊、Session Key（遮罩輸入）、Org ID 輸入欄位。
  4. **一般**：「開機時自動啟動」核取方塊。

## 8. 驗收標準（Acceptance Criteria）

1. 啟動後桌面出現圓形 GIF 小幫手，置頂、不佔工作列。
2. 點擊主圓圈展開衛星（含展開/收合動畫）；再點一次收合。
3. 點擊衛星能成功開啟對應目標（執行檔、資料夾、網址各驗證一項）；目標失效時顯示錯誤而非閃退。
4. 透過齒輪 → 設定視窗更換 GIF，主圓圈立即更新。
5. 在設定中新增/刪除連結後，衛星數量與內容即時反映。
6. 拖曳主圓圈後結束程式再啟動，位置保留。
7. 系統匣右鍵選單可隱藏/顯示小幫手、開啟設定、結束程式。
8. 勾選開機自啟後，登錄檔 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` 出現對應項目；取消勾選則移除。
9. 刪除或損毀 config.json 後啟動，程式以預設值正常執行。
10. 在設定填入有效 Session Key／Org ID 並啟用後，主圓圈外圍出現三層圓環，弧長與顏色對應實際用量；懸停顯示百分比與 reset 倒數。
11. 填入無效 Session Key 時，三環轉灰、懸停顯示錯誤訊息，程式不閃退且 GIF 動畫照常播放。
12. 停用用量顯示後，圓環消失且不再發出網路請求。
13. 選擇含多個 GIF 的資料夾後，主圓圈依設定間隔隨機輪播且不連續重複；資料夾無 GIF 時顯示預設圖案。
